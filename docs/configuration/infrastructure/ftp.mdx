# FTP browsing

FTP Browsing section lets you configure how the FTP connections are established, and results are displayed.

SafeSquid can provide support for browsing FTP Servers, anonymously.

This can be useful when the client (some of the older web-browsers) applications lack the support to directly access FTP sites.

Such Internet Clients may also set SafeSquid as their HTTP/1.1 Proxy to access FTP sites.

The settings made here apply to all requests for ftp: // URLs.

## Global Settings

Configure section-wide settings that apply to all entries.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| **Passive mode** | `BOOL` | `ftp_enable` | When you are behind a firewall, it prevents the FTP server from opening a data connection to you.

Thus, it prevents the FTP transfers.

Enabling Passive mode solves this by opening up both (Command and Data) channels from your side.

So, it is recommended to use Passive mode when you are behind a firewall. |
| **Timeout** | `INT` | `15` | The timeout to wait for a response to commands sent to the FTP server. |
| **Username for anonymous login** | `STRING` | `anonymous` | Specify the username to be used when none is given in the URL, explicitly |
| **Password for anonymous login** | `STRING` | `noone@nowhere.com` | The password to be used when none is given in the URL, explicitly |
| **Sort order** | `STRING_SELECT_ONE` | `order` | Specify the order by which FTP directory listings should be sorted. |
| **Sort field** | `STRING_SELECT_ONE` | `field` | Specify the field by which FTP directory listings should be sorted. |
