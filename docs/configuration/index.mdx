# Configuration Reference

This section provides comprehensive documentation for all SafeSquid configuration sections.


## Infrastructure

- **[Accelerators](infrastructure/accelerator)** - Use accelerators to make your internet faster and more efficient. They help save resources and make websites load quicker.
- **[Access restrictions](infrastructure/access)** - Control who can and cannot use SafeSquid's services. You can allow or block access for specific people or groups of people.
- **[Proxy chain](infrastructure/forward)** - The Proxy chain section allows you to forward requests through another proxy selectively, SOCKS4 or SOCKS5 firewalls.
- **[FTP browsing](infrastructure/ftp)** - FTP Browsing section lets you configure how the FTP connections are established, and results are displayed.
- **[System configuration](infrastructure/general)** - Use 'System configuration' to tune various parameters with respective network infrastructure.
- **[Integrate LDAP](infrastructure/ldap)** - Integrate with Microsoft Active Directory or OpenLDAP.  
- **[Network settings](infrastructure/network)** - Configure the basic Network Infrastructure.
- **[Subscription](infrastructure/subscription)** - View your current subscription and license information here.


## Configuration

- **[Real time content security](configuration/antivirus)** - Configure security settings to keep your network safe and protected.
- **[Application Setup](configuration/infrastructure)** - Set up your internet security system.
- **[Restriction Policies](configuration/policymgr)** - Set detailed rules and restrictions for different users or groups of users.
- **[Custom Settings](configuration/profilesdef)** - Customize configuration to suit your internet access policies.


## Profiles

- **[Application Signatures](profiles/application-signatures)** - Manage how SafeSquid identifies and categorizes different applications and websites that people use on your network.
- **[Categorize Web-Sites](profiles/category-editor)** - Use Categorize websites to Manage Web-Site Categorization.
- **[Content Signatures](profiles/content-signatures)** - This Section allows you to manage SafeSquid's response profiling engine.
- **[External applications](profiles/external)** - The external feature lets you use any program or script to examine and process file contents.
- **[Request Types](profiles/req-profiles)** - Use Request Profiler to organize and categorize requests based on what people are trying to access.
- **[Response Types](profiles/resp-profiles)** - Use Response profiles to organize and categorize content based on what websites send back to users.
- **[Templates](profiles/templates)** - Custom templates can be very helpful when:
- **[Time Profiler](profiles/time-profiles)** - Use Time Profiler to set up rules about when people can access the internet.


## Policy Management

- **[Suggested Profiles](policy-management/buitins)** - List of Built-in Profiles
- **[Privacy control](policy-management/filter)** - Setup controls to ensure safe information exchange between users and remote websites.
- **[Speed Limits](policy-management/limits)** - Set data transfer rate (speed or QoS) limits.
- **[Access Profiles](policy-management/profiles)** - Use Access Profiles to set up rules about who can access what on the internet.


## Performance

- **[Caching](performance/cache)** - You can add/remove cache stores in this section, and configure: Global cache options
- **[Prefetching](performance/prefetch)** - Use Prefetching to make websites load faster for users.


## Security

- **[Clam antivirus](security/clamav)** - With this antivirus feature you can detect and block files containing viruses known to the ClamAV daemon scanner.
- **[DNS Blacklist](security/dnsbl)** - DNS Blacklist
- **[Text analyzer](security/keywords-filtering)** - Text Analyzer helps you find and block websites that contain inappropriate content by looking for specific words.
- **[Redirect](security/redirect)** - Create policies to redirect requests for a URL to another URL.
- **[HTTPS Inspection](security/sslcert)** - This section allows you to configure the overall operation to handle scanning of connections under HTTPS (HTTP over SSL).
